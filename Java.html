MemoryLeak

  • 在 webapp 環境下, tomcat 才是 jvm 的一個 application, 所以當你在 webapp 中使用 daemon thread, 連續的 deploy and undeploy 動作下, daemon thread 並不會被關閉因為 tomcat 並還沒有關閉, 而造成 PermGen memory leak.

若啓動 daemon thread 的是自己有下列解法: 1. http://stackoverflow.com/a/792010/474002 (用 context listener 聽取 undeploy 事件把 thread terminate) 2. http://stackoverflow.com/a/8663302/474002 (使用 Runtime.getRuntime().addShutdownHook(....) 把 thread 加入 shutdown hook) 3. http://stackoverflow.com/a/11902443/474002 (用ThreadGroup 把所有 thread 掃出來, 然後看哪個是 daemon 把它幹掉)

Singleton
  • Singleton design pattern in java
    http://howtodoinjava.com/2012/10/22/singleton-design-pattern-in-java/ 
public class DemoSingleton implements Serializable {
  private static final long serialVersionUID = 1L;

  private DemoSingleton() {
  	// private constructor
  }

  private static class DemoSingletonHolder {
  	public static final DemoSingleton INSTANCE = new DemoSingleton();
  }

  public static DemoSingleton getInstance() {
  	return DemoSingletonHolder.INSTANCE;
  }

  protected Object readResolve() {
  	return getInstance();
  }
}
