Persistence

JPA JPQL

  • JPA 和 Validation 基本上是獨立的兩個東西, Hibernate 會去自動 check Validation 的 constraint 但是有些時候 constraint 只想用在特定的 persist 操作 (像是 saveNew), 有時候只是想要更新某個欄位, 並不需要 validate 該 constraint, 這時候就只能用 native query 繞過該 validation
  • 測試當中若有 Transactional 的操作, 如果primary key 不是 auto increment 都加上 flushAndClearEntityManager 然後在用 native select 驗證結果

Notes

  • 舊的 code 可以採用 JDBC Template 簡化
  • JSON 發生問題可以用 WebApplicationException 拋出 JSON 格式的異常訊息
  • 有一個全域的 .yaml 然後各自的 DAO 要測試可以用 DbUnit 準備自己要測試的資料
  • DAO 寫資料進資料庫前會先跑 Validator
  • 所謂有 Jesey context 的意思就是繼承 AbstractWebServiceTest, 他可以幫 Resource 的測試準備 http 環境
  • Spring 的設定在 config/ 下面
  • 用 JPA 的時候就用 JPQL 做 query, 用 native query 會回傳 Object[][], 就沒用到 orm 的特性
