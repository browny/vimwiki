Effective Java

  1. 靜態工廠方法 (返回類的實例的靜態方法)
    • 優點:
      1. 與建構子不同的地方在於, 他 method name 有意義, 而非單純類的名稱
      2. 搭配 singleton, 不必像建構子一樣每次都創建一個新的物件返回
      3. 可以返回其類的任何子類型物件 (Service Provider Framework: Service Interface, Service Provider Interface, Provider Registration API, Service Access API)
      4. 建構參數化類的實例時, 使代碼更簡潔 
Map<String, List<String>> m = new HashMap<String, List<String>>();

假設 Map 提供靜態工廠方法:
public static <K, V> HashMap<K, V> newInstance() {
	return new HashMap<K, V>();
}

就可以簡化為:
Map<String, List<String>> m = HashMap<K, V>();
  • 缺點:
    1. 若沒有 public/protected 建構子, 就無法被 subclass
    2. 與其他靜態方法實際上沒有區別
