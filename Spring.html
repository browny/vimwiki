Note

Spring 怎麼決定他要管理哪些 class (bean)

Spring 的主要入口在 SpringAppConfig.java 這個設定檔, 由 src/main/webapp/WEB-INF/web.xml Spring context configuration 裡面設定 (org.springframework.web.context.support.AnnotationConfigWebApplicationContext) ContextRefreshedListener 把 Spring context share 給非 Spring 管理的 class, 因此非 Spring 管理的 class 可以使用 getApplicationContext 或是 @Inject BeanFactory 來取的 Spring Bean

這個 SpringAppConfig 會去掃底下的 package 和所有的 @Configuration 的 class, 然後這些 class 再去掃下面 @ComponentScan 的 package, @Import 進來的 class, 和下面 define 的 @Bean

什麼是 Bean, 下面這 4 種 annotation 就是 bean:

@Named, @Service, @Repository

@Bean
public xxx XXX() {} // 參考 src/main/java/com/fanzytv/config/DatabaseConfig.java, 代表這個方法會獲得一個 bean

宣告為 Bean 以後, 這些物件就歸 Spring 管理, 就可以用 @Inject, @Named 注入該物件

Spring
@Component: Indicates a auto scan component

@Repository, @Service, and @Controller serve as specializations of @Component for more specific use cases

@Service: Indicates DAO component in the persistence layer.
@Repository: Indicates a Service component in the business layer
@Controller: Indicates a controller component in the presentation layer.
@Bean: Annotation to be applied to methods that create beans in a Spring context

JSR 330
@Inject instead of Spring’s @Autowired to inject a bean.
@Named instead of Spring’s @Component to declare a bean.

@Transactional
Any RuntimeException triggers rollback, and any checked Exception does not.
Only unchecked exceptions (that is, subclasses of java.lang.RuntimeException) are rollbacked by default
