什麼是 Session?

1. 某人輸入正確密碼登入 2. 伺服器產生一個新的、唯一的、超難猜的 session_id，如果很好猜就會提高被人冒認的機會， 例如不要用 1, 2, 3,.. ，而要用類似 !@@a#$%:-)XP>_< 這些超難猜的東西。 3. 伺服器記著這個 ID 和它的最近訪問時間（假設這個 session_id='id1'）： Session 清單： id1 => 2005/07/15 11:30:00

4. 伺服器透過 Cookie ，把 session_id='id1' 儲存在客戶端。 5. 每次客戶端訪問伺服器，都會把 session_id='id1' 送到伺服器。 6. 伺服器收到 session_id='id1' ，就在清單內找一找是否真的有名叫 'id1' 的 Session， 而且透過最近訪問時間，就可以知道他閒置了多久，如果太久的話可以在清單刪除這個 session， 強逼他重新輸入密碼登入。
