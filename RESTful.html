Notes

  1. URL 不能有 verb 是一個誤區, 把所有的動作都放到以 action 為名的 QueryParam 只會讓 URL 有過多的職責
  2. 可以把執行動作的 URL 想像成 linux 的指令, 把越容易變動的特性放在網址的越尾端
  3. PathParam, QueryParam 就像是 filter，決定最後結果對外呈現的樣貌
  4. 好的 URL 設計應該要可以讓 API 文件相當簡潔易懂
  5. 一個好的 API 是能夠讓開發者馬上了解該做些什麼，以及他能做些什麼
